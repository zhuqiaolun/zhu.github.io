发布Java程序为Windows服务 （修改配置参数，新增对 jvm 的配置）

原创 2020-06-23 15:57:53 工具 Java 0 字数统计: 206(字) 阅读时长: 1(分)

前面是使用的配置会导致JVM不断重启，需要加JVM参数以及设置，同时设置服务开机启动；

配置文件

新增配置参数如下：

wrapper.jmx=false
wrapper.on_exit.0=SHUTDOWN
wrapper.on_exit.default=RESTART
wrapper.ntservice.interactive = true
# 服务开机启动
wrapper.ntservice.starttype=AUTO_START
wrapper.tray = true
# 监测JVM死锁
wrapper.java.monitor.deadlock = true
wrapper.java.monitor.heap = true
wrapper.java.monitor.gc.restart = true
# Java Heap 初始化大小(单位：MB)
wrapper.java.initmemory=128
# Java Heap 最大值(单位：MB)
wrapper.java.maxmemory=128
# 32/64位选择，true为自动选择
wrapper.java.additional.auto_bits=TRUE

检测JVM参数

可以用jps+jmap监测即可
jps:

jmap -heap xxxxx
